# Complete logic for multiple sitemaps

## Goal

Use:

* `sitemap.xml` → main sitemap index
* `sitemap-jobs.xml` → all active job pages
* `sitemap-pages.xml` → homepage + static pages + category pages
* `sitemap-blogs.xml` → blog pages

---

# 1. Main idea

Your pipeline already does:

...
* Phase 4 → generate HTML pages

After that, add:

* Step 5 → generate all sitemap files
* Step 6 → optionally ping Google

---



# 3. What each sitemap should contain

## A. `sitemap.xml` = sitemap index

This file does **not** contain job URLs directly.

It only contains links to the other sitemap files.

Example:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<sitemapindex xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <sitemap>
    <loc>https://findjobsinfinland.fi/sitemap-jobs.xml</loc>
    <lastmod>2026-04-06T18:00:00+03:00</lastmod>
  </sitemap>
  <sitemap>
    <loc>https://findjobsinfinland.fi/sitemap-pages.xml</loc>
    <lastmod>2026-04-06T18:00:00+03:00</lastmod>
  </sitemap>
</sitemapindex>
```

<sitemap>
  <loc>https://findjobsinfinland.fi/sitemap-blogs.xml</loc>
  <lastmod>2026-04-06T18:00:00+03:00</lastmod>
</sitemap>
```

---

## B. `sitemap-jobs.xml`

This contains:

* all active job detail pages only

Each entry should look like:

```xml
<url>
  <loc>https://findjobsinfinland.fi/jobs/construction-and-trades/electrician-finland-9b0c490e</loc>
  <lastmod>2026-04-06T18:00:00+03:00</lastmod>
</url>
```

### Rules:

* include only active jobs
* exclude expired jobs
* exclude deleted jobs
* exclude broken URLs
* use final public URL only
* no duplicate URLs

---

## C. `sitemap-pages.xml`

This contains non-job pages, for example:

* homepage `/`
* main jobs page `/jobs/`
* category pages like `/jobs/construction-and-trades/`
* about page
* contact page
* privacy page
* any other real static pages

Example:

```xml
<url>
  <loc>https://findjobsinfinland.fi/</loc>
  <lastmod>2026-04-06T18:00:00+03:00</lastmod>
</url>
<url>
  <loc>https://findjobsinfinland.fi/jobs/</loc>
  <lastmod>2026-04-06T18:00:00+03:00</lastmod>
</url>
<url>
  <loc>https://findjobsinfinland.fi/jobs/cleaning-and-facility-services/</loc>
  <lastmod>2026-04-06T18:00:00+03:00</lastmod>
</url>
```

# 5. Expired jobs logic

This is very important.

## Rule:

If a job is expired, it must be:

* removed from job folder
removed from job card in both index and jobs.html
* removed from `sitemap-jobs.xml`


### So the check should be:

Before adding a job to sitemap:

```text
if job is active and public and not expired:
    include in sitemap-jobs.xml
else:
    skip
```

---

# 6. Datetime logic for `lastmod`

Since your site updates multiple times per day, use:

```text
YYYY-MM-DDTHH:MM:SS+03:00
```

Example:

```xml
<lastmod>2026-04-06T18:00:00+03:00</lastmod>
```

## Best practice for your case:

Use the current pipeline run time for all sitemap entries generated during that run.

That means:

* simpler
* consistent
* good enough for Google

---

# 7. Duplicate prevention logic

Before writing URLs into any sitemap:

* normalize URLs
* remove duplicate entries
* ensure trailing slash policy is consistent
* ensure `.html` is not mixed with non-`.html` version unless your site truly uses it as canonical format

## Example:

Choose one format only:

Good:

```text
https://findjobsinfinland.fi/jobs/other/a-barber-seinäjoki-6dcf23c8
```

Do not also include:

```text
https://findjobsinfinland.fi/jobs/other/a-barber-seinäjoki-6dcf23c8.html
```

unless that is your only real public URL.

---




# 13. Simple XML rules

For every sitemap file:

* start with XML declaration
* use correct sitemap namespace
* escape XML special characters in URLs if needed
* keep valid UTF-8
* no script tags
* no browser extension junk
* no malformed tags
