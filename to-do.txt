now update the JOB CATEGORY DETERMINATION LOGIC

Overview:
Job categories are determined using a rule-based scoring system supported by keyword matching.
The system prioritizes accuracy, consistency, and explainability.
AI is only used as a fallback when the system is uncertain.

--------------------------------------------------

1. KEYWORD MATCHING (INITIAL DETECTION)

Each job category contains a predefined list of English keywords.

The system scans:
- Job title
- Job content

Rules:
- A category is valid only if multiple keywords match
- Minimum requirement: at least 2–3 relevant keyword matches
- A single keyword match is NOT sufficient

Example:
"Software Developer, frontend, backend, full stack"
→ Matches multiple keywords → Category: it-and-tech

If no category meets the minimum threshold:
→ Assign category: "other"

--------------------------------------------------

2. SCORING-BASED CLASSIFICATION

All categories are scored. The highest score wins.

Scoring factors:
- Title exact match: +12
- Title partial match: +8
- Content keyword match (repeated): +3 each
- ESCO / occupation mapping: +20
- Source taxonomy match: +15
- Negative keyword conflict: -8
- Generic keyword only: +1

--------------------------------------------------

3. PRIORITY OF SIGNALS

Importance order:
1. Structured data (ESCO / occupation codes) → Highest
2. Job title → Very high importance
3. Job content → Supportive only

--------------------------------------------------

4. TITLE DOMINANCE RULE

The job title is the strongest signal.

Examples:
Cleaner → cleaning-and-facility-services
Practical Nurse → healthcare-and-social-care
Sales Manager → business-and-finance
Waiter → food-and-restaurant
Carpenter → construction-and-trades

If the title clearly indicates a category, it overrides weaker signals.

--------------------------------------------------

5. NOISE HANDLING (JOB CONTENT)

Ignore:
- Generic company descriptions
- Soft skills (teamwork, communication, etc.)
- Irrelevant text

Only count relevant job-specific keywords.

--------------------------------------------------

6. CATEGORY SELECTION

All categories compete.
The category with the highest score is selected.

Example:
it-and-tech → 31
business-and-finance → 11

→ Selected: it-and-tech

--------------------------------------------------

7. CONFIDENCE-BASED DECISION

Strong match:
Top1 = 31, Top2 = 11 → Clear winner → Select Top1

Weak/ambiguous match:
Top1 = 18, Top2 = 17 → Too close → Low confidence

--------------------------------------------------

8. AI USAGE (OPTIONAL TIE-BREAKER)

AI is NOT used for primary classification.

Use AI only when:
- Scores are too close
- Confidence is low
- Multiple categories overlap

Rules for AI:
- Only choose from top candidate categories
- Prioritize job title over content

--------------------------------------------------

9. FALLBACK RULE

If:
- No category meets keyword threshold
- Or scores are too weak

→ Assign category: "other"

--------------------------------------------------

CORE PRINCIPLES

- Do NOT rely on a single keyword
- Do NOT select the first match
- Always use scoring and comparison
- Prioritize title over content
- Use AI only as a tie-breaker
- Prefer correct broad category over wrong specific one
